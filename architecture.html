MediaLens — Agent Architecture

Two LLM agents, one shared tool set, one exclusive web-search tool.

🔎
Research Agent
src/agent.py · Ask tab
Purpose

Answers open-ended topic questions by searching the internal database and synthesizing how left, center, and right outlets frame the same story.

Tool-Calling Loop
Iteration max 10 rounds
User query → LLM decides next tool
↓ finish_reason = tool_calls
Execute tool → result appended to context
↓ repeat until done
→ finish_reason = stop → final response
Mandatory First Step
1
Check data freshness — always calls get_database_stats before anything else
Tools (7 shared)
🔍
search_stories Cosine-similarity search over cluster headlines
📄
get_story_detail All articles + analysis for a cluster
⚖️
get_lean_breakdown Articles grouped by left / center / right
🕒
list_recent_stories Latest clusters, optionally filtered by lean
📈
get_trending_topics Most cross-spectrum stories by coverage breadth
🗄️
get_database_stats Article/cluster counts and last fetch time
📡
list_sources All 9 configured RSS sources with their political lean
Required Output Structure
## Topic Overview
## What Sources Agree On
## Where Sources Disagree
## How Each Lean Frames It
## Notable Omissions
✅
Fact-Check Agent
src/fact_check.py · Fact Check tab + story detail
Purpose

Verifies a specific claim using both the internal article database and live web search. Follows an adversarial pattern: must find evidence for and against before delivering a verdict.

Mandatory Steps (enforced by system prompt)
1
Data freshness check — get_database_stats
2
Topic search — search_stories to find related clusters
3
Lean breakdown — get_lean_breakdown on relevant clusters; explicitly look for support and contradiction
4
Live web search — search_web (DuckDuckGo) for real-world evidence: official statements, statistics, primary sources
5
Optional detail drill-in — get_story_detail if specific wording needs checking
6
Deliver structured verdict
Tools (7 shared + 1 exclusive)
Inherits all 7 research agent tools, plus:
🌐
search_web DuckDuckGo live web search via ddgs. Returns title, URL, snippet for up to N results. Used to verify facts outside the internal database.
Context Injection (story detail page)
When invoked inline from a story page, the headline is prepended so the agent prioritises the right cluster:

"Context: <story headline>
Claim to fact-check: <user claim>"
Possible Verdicts
CONFIRMED DISPUTED MISLEADING UNVERIFIABLE
** VERDICT — one-sentence explanation
** Evidence for the claim
** Evidence against / complicating context
** What each side emphasises (L / C / R)
** Confidence assessment
Shared Infrastructure
LLM
DeepSeek v4 Flash via OpenRouter (OpenAI-compatible API). Temperature 0.2. Up to 10 tool-call iterations per query.
RAG (no vector store)
search_stories batch-encodes all cluster headlines at query time with all-MiniLM-L6-v2 and ranks by cosine similarity. No Pinecone/Weaviate — fast enough for ≤350 clusters.
Storage
All tool results read from SQLite (data/medialens.db). Tool outputs serialised to JSON and returned as role: "tool" messages in the conversation.
